## データの前処理と特徴量エンジニアリング
<br>

予測精度を向上させるため、乗客データを以下の3つのグループ（意味合い）に分類し、論理的なアプローチで特徴量を作成・加工しました。  
<br>

### 個人の属性情報（Age, Sex, Title）


・Title（敬称）の抽出:

Nameカラムから正規表現を用いて「Mr」「Miss」「Mrs」などの敬称を抽出。DrやRevなどの希少な敬称はRareとして統合し、社会的地位による生存率の差を捉えやすくしました。  

・カテゴリの数値化:

Sex（性別）を数値（0/1）にマッピング。  

・Ageの欠損補完とビン分け:

欠損値は全体の平均値で補完。さらに連続値であるAgeを5段階のカテゴリに分類（Binning）し、年齢層によるパターンの学習を容易にしました。  

<br>

### 家族や同伴者の情報（SibSp, Parch）
  

・FamilySizeの作成: 

SibSp（兄弟・配偶者数）とParch（親子数）に自分自身（+1）を合計し、家族の総人数を算出。  

・IsAloneフラグの導入: 

FamilySizeが1の場合を「単身（IsAlone=1）」とし、同伴者の有無を明確な特徴量として追加しました。  

<br>

### チケットと乗船状況（Fare, Embarked, Cabin）

 

・Embarkedの補完と数値化: 

欠損値（2件）を最頻値（'S'）で補完し、数値にマッピング。  

・Fareのビン分け: 

Fare（運賃）の分布の偏りを軽減するため、4段階のカテゴリに分割（qcutによる分位点分割）しました。  

・Cabinの除外: 

欠損値が全体の約77%と非常に多いため、予測モデルからは削除しました。
